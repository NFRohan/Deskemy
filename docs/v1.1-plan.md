# Deskemy v1.1 — Plan

Target: the **first feature release after the 1.0.1 public launch** (app version
`1.1.0`). ROADMAP.md's `v1.1–v1.5` were the *pre-1.0* development sprints, all
folded into the first release; public versioning launched at 1.0.1, so this is
the next one up.

**Theme:** protect the user's data, keep installs current, and small daily-use
polish. Nothing here touches the player's core or the airspace/compositor rules
— all additive.

Status legend: 🔑 headline · ⏳ planned · 💡 open question to resolve before building.

---

## 1. 🔑 Backup & restore

**Why it's the headline.** Everything Deskemy records — progress, bookmarks,
tags, favorites, career tracks, stats, course art — lives in one local SQLite
database plus a thumbnail cache under the data dir. There's no cloud by design,
so backup is entirely on the user. Losing that folder loses everything. Make
saving and restoring it a one-click thing. (This is local-only users' #1 fear.)

**Scope**
- **Export** → a single archive containing `deskemy.db` + `config.json` +
  `thumbnails/`, with a small manifest (app + schema version, timestamp).
- **Restore** → pick an archive, confirm, swap the data in. App restarts or
  hot-reloads the library.
- **Not included:** the video files themselves (referenced in place, never
  copied). Restore only rebuilds the *index*. If the videos moved on disk,
  rename-detection (`content_hash`, already built in v1.4) re-links them on the
  next scan — document this clearly so users aren't surprised.
- **UI:** Settings → Data → *Export backup* / *Restore backup*.

**💡 Open questions**
- **Replace vs merge.** Start with **full-replace** (simplest, unambiguous).
  Merge across two machines is conflict-prone (same course id, diverging
  progress) — defer; UUIDv7 keys keep the door open, same as the cut CRDT sync.
- **Archive format.** Plain `.zip` (universal, inspectable) vs a branded
  `.deskemybak` (just a renamed zip). Lean `.zip`.
- **Schema drift.** Manifest carries the schema version; restoring a *newer*
  backup into an *older* app should refuse with a clear message. Restoring an
  older backup runs the normal migrations.
- **Auto-backup** (keep last N to a chosen folder) — nice, but v1.2+.

## 2. 🔑 In-app updates (auto-updater)

**Why now, and why it can't wait.** There are real users (13+ downloads at first
release) and an installed copy currently has no way to learn a newer version
exists — so every release strands whoever's on the old one. Any update mechanism
only works *from the version that first ships it*, so the longer this slips, the
more stranded cohorts pile up. Ship it in v1.1 regardless of feature order.

> The 1.0.x users are a one-time stranded cohort — unreachable automatically
> (their build has no checker). Only a release announcement + GitHub "Watch →
> Releases" notifications nudge them forward this once. From v1.1 on, it's automatic.

**Flow (the standard Tauri v2 updater — notify, then user-initiated download):**
- On launch, check a `latest.json` manifest for a newer version.
- If newer → show a **dismissible banner** ("Deskemy vX.Y is available"), and
  surface the same in **Settings → Updates**.
- User accepts — from the banner *or* the Settings button — → **download +
  install**, then relaunch. No silent/background installs; the user always opts in.

**Stack:** `@tauri-apps/plugin-updater` (+ `tauri-plugin-updater`), `check()` →
`downloadAndInstall()`. `latest.json` hosted on the GitHub release.

**Signing (required, but free):** updates must be signed with an **ed25519 key**
(`tauri signer generate`) — public key in `tauri.conf.json`
(`plugin.updater.pubkey`), private key kept as a repo secret and used at build to
emit `.sig` files. This is separate from — and far cheaper than — the Authenticode
code-signing cert (see below).

**💡 Open questions / caveats**
- **Portable can't self-install** (loose exe + bundled `libmpv-2.dll`). Portable
  gets a **notify-only** banner that links to the release; auto-install is
  installer-only. Both share the same "is there a newer release?" check.
- **SmartScreen per update.** Until a code-signing cert is in place, each
  downloaded update installer may re-trigger SmartScreen/UAC — works, just not
  silent; reputation eases it, the cert removes it.
- **Manifest generation.** Each release must emit `latest.json` (version, notes,
  signature, asset URL) — a small script now, or `tauri-action` once CI exists.
- **Update path.** Confirm the NSIS `currentUser` install-mode update replaces
  cleanly end-to-end (incl. the bundled DLL) before shipping.

## 3. ⏳ Sleep timer  *(quick win)*

Stop playback after N minutes, or "**after this lecture**." Lives in the player
menu; optional shortcut. Pure frontend + a pause call — no schema change.

---

## 4. ⏳ Per-lecture notes  *(quick win, medium)*

A notes panel in the player, alongside Resources (`R`) — freeform text/markdown,
saved per lecture, timestamp-insert optional.
- New `notes(lecture_id, body, updated_at)` table (schema bump). Not bookmarks —
  those are timestamped markers; notes are freeform prose.
- **💡** Index in FTS for search? Defer to a later release; v1.1 notes can be
  non-searchable to keep it small.

---

## 5. ⏳ Richer library filters  *(quick win)*

Finish the roadmap's "better library filters": recently watched / never started /
finished / in progress / favorites / **tags** / **career track**. Mostly
client-side (the data's already loaded); extends the existing status chips.

---

## Sequencing

1. **Backup & restore** — headline value, self-contained.
2. **In-app updates** — time-sensitive infra; the earlier it lands in v1.1 the
   fewer future stranded cohorts. Can go in parallel with #1.
3. **Sleep timer** — tiny, ride along.
4. **Library filters** — client-side, low risk.
5. **Per-lecture notes** — schema + UI, slightly more.

## Also on the table (not scheduled here)

- **Code signing** — the app ships unsigned, so SmartScreen warns on first run
  *and* on each auto-update. A cert removes both; it's the natural companion to
  the auto-updater. Track as its own (paid) decision.
- **CI** (fmt/clippy/test + svelte-check + eslint) — cheap regression insurance
  now that outside machines run it; a svelte-check-only type bug slipped past
  `npm run build` once already. Also the clean home for auto-generating the
  updater's `latest.json` per release (`tauri-action`). Carried from ROADMAP.md.
