## Queued

### ⏳ Clean up leftover auto-update installers in %TEMP%

**Problem.** After every installer-path auto-update, the downloaded NSIS
installer (~37 MB) is left behind in the user's temp dir. It's an upstream
`tauri-plugin-updater` behavior (verified in 2.10.1), not our code:

- `downloadAndInstall()` writes `Deskemy_<ver>_x64-setup.exe` to
  `%TEMP%\Deskemy-<ver>-updater-<rand>\Deskemy-<ver>-installer.exe`.
- The temp dir is created with `.keep()` (opts out of auto-delete).
- The installer file is wrapped in a `tempfile::TempPath` (delete-on-drop),
  BUT `install_inner` launches the installer via `ShellExecuteW` and then calls
  `std::process::exit(0)` — which does NOT run destructors, so the cleanup guard
  never fires. The `process::exit` is necessary (release file locks so NSIS can
  overwrite the app), so this can't be fixed from our call site.

Result: one ~37 MB installer + its `Deskemy-<ver>-updater-*` folder leaks per
update. Low severity (it's in %TEMP%; Storage Sense eventually reclaims it) but
real, and compounds across releases. The **portable** path is clean (it only
opens the release page — no Deskemy-managed download).

**Fix.** A cheap startup sweep, mirroring the existing startup auto-reconcile
thread in `src-tauri/src/lib.rs` (the `library:changed` one):

- On launch, off the main thread, scan `std::env::temp_dir()` for
  `Deskemy-*-updater-*` directories and best-effort remove them (ignore errors
  from locked files).
- **Safe** because by the time the sweep runs the app IS the post-update
  version, and the installer that spawned it has essentially finished — those
  dirs are stale. Any file still locked (rare race: installer exe not yet
  exited) just fails silently and gets swept on the next launch.
- ~15 lines. Scope the glob to our app prefix so we never touch unrelated temp
  files. No dependency on the plugin internals beyond the well-known prefix.

**Testing:** light — trigger an update (or drop a dummy `Deskemy-x-updater-*`
dir in %TEMP%), relaunch, confirm it's gone; confirm a locked file is skipped
without erroring.
